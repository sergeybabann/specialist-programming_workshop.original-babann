Зачем и какой брать Тип данных?

Переменная которая складывает числа должна иметь тип данных

Тип данных int хранит точную информацию о числе, а тип double (число с плавающей точкой) хранит до 14 знаков после запятой. Если складывать число со знаками более 15 в double, то можно потерять знаки после запятой. Будет не точная сумма. Для точности необходимо использовать тип данных long, и может быть делать даже с проверкой.

Тип данных Big Integer - он позволяет хранить точные числа любого размера, но он работает медленне, чем long и double, и будет расходовать больше памяти.

Работаем в package day11 class day11.java

class day11.java

Ввод данных с клавиатуры - мы должны добраться до объекта, который отвечает за чтение или отправку данных.
import java.util.Scanner; - подключение библиотеки вверху документа java
Scanner scan = new Scanner(System.in); // Scanner - тип объекта, scan - имя объекта, in - имя объекта. class day11.java

У нас есть объект in, который позволяет работать с клавиатурой. Мы его обернули в объект scan, который предоставит нам удобные методы работы с клавиатурой. Но мы будем пользоваться объектом scan, который является потоком и позволяет читать данные с клавиатуры также, как и объект in, но в более удобном варианте. И на этом объекте scan мы будем использовать метод nextInt() чтобы прочитать нужное нам значение.
scan.nextInt();

Вызов метода nextInt() вернет целое число.

Для того чтобы введенное значение запомнить нужно создать переменную и положить туда эти данные.

Работаем в package day11 class day11_2.java

class day11_2.java

Заменили int на double, и теперь можем в запуске программы вводить в консоли также числа с плавающей точкой. Поэтому можно сказать, что double предоставляет больше возможностей в этом смысле. Можно вводить в консоль целые и дробные числа. Но на данный момент настроек среды разработки ставить надо запятую вместо точки потому, что точку не пускает консоль.
(можно поменять в настройках среды разработки на точку).

!!!Возникает контрольный вопрос - Зачем нужен int , если double дает больше возможностей?

Мы имеем программу из 2-ух этапов:
Run day11_2.java
Открывается консоль и мы начинаем вводить в консоль значения для нашей программы. 1 этап) Ввести 3 числа double, результат - суммирование этих чисел
2 этап) Ввести строку String (можно имя), результат - получаем введенное значение String

Выполняем еще раз Run day11_2.java

На 1-ом этапе программы вводим 3 значения double

На 2-ом этапе программы вводим 3 значения String через пробел - Сергей Андрей Виктор

Результат - будет выведено 1-ое значение String в порядковом номере.
