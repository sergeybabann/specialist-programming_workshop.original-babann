Зачем и какой брать Тип данных?

Переменная которая складывает числа должна иметь тип данных

Тип данных int хранит точную информацию о числе, а тип double (число с плавающей точкой) хранит до 14 знаков после запятой. Если складывать число со знаками более 15 в double, то можно потерять знаки после запятой. Будет не точная сумма. Для точности необходимо использовать тип данных long, и может быть делать даже с проверкой.

Тип данных Big Integer - он позволяет хранить точные числа любого размера, но он работает медленне, чем long и double, и будет расходовать больше памяти.

Локальные переменные

Любая переменная (объект), которая сделана внутри метода называется - локальная переменная. Внутри метода - это означает между открывающей и закрывающей скобками метода.
Например, метод main:
public static void main (String[] args) {
// между открывающей и закрывающей скобками метода
}

1) До первого своего использования локальной переменной мы обязаны присвоить ей начальное значение. Если этого не сделать, то это является ошибкой.

Объявление переменной и присвоение ей значения:
double a;
a = scan.nextDouble;

2) Вариант. Переменную можно инициализировать сразу же как только мы ее объявили:
double a = 0;

Во всех компилируемых языках программирования всегда переменную нужно явно создать.

  • создать: double a;
  • инициализировать, дать начальное значение: double a = 0;

Что можно использовать для инициализации переменной?

  • Можно использовать константное значение. Например: любое число
  • Можно использовать другую переменную.
    Одна переменная равна другой переменной:
    summa = a; Переменная summa равна переменной a. То есть мы значение переменной a присвоили значению переменной summa
Жизненный цикл и область видимости локальных переменных

Как уже ранее говорилось локальная переменная существует только внутри метода. И все операции связанные с ней можно выполнять только после ее объявления, ниже по странице.

Хорошим тоном является объявление всех переменных в начале файла.

Жизненный цикл переменных от момента ее объявления до закрывающих скобок метода.

Имена переменных должны быть всегда уникальны. Невозможно использовать одно и то же имя переменной 2 раза внутри области видимости метода.

Арифметические операции

Оператор -
Этот оператор изменяет число с положительного на отрицательное и наоборот.

int a, b = 5, c = -2;
a = -c; // - operator (this is remainder from division)
System.out.println(a); // result 2

Переменная c равна отрицательному числу -2
Далее мы ставим оператор - для переменной c
Эта операция изменяет отрицательное число -2 на положительное число 2
И приравниваем это число 2 переменной a
Запускаем нашу программу в терминале и получаем результат число 2

Оператор +

int a1, b1 = 5, c1 = -2;
a1 = +c1;
System.out.println(a1); // result -2

Запускаем нашу программу в терминале и получаем результат отрицательное число -2

Ничего не изменилось. Унарный оператор + не произвел никаких изменений со значением переменной с1

Операция сложения 2-ух чисел

int a2, b2 = 5, c2 = -2;
a2 = -b2 + c2;
System.out.println(a2); // result -7

Складывается -5 и -2, результат сложения отрицательное число -7

2) Вариант сложения. Операция сложения 2-ух чисел

int a3, b3 = 5, c3 = -2;
a3 = +(c3 - b3);
System.out.println(a3); // result -7

Складывается 2 числа. Отрицательное число -2 и отрицательное число -5
Запускаем нашу программу и получаем результат отрицательное число -7

Операция вычитания, затем смена отрицательного числа на положительное

int a4, b4 = 5, c4 = -2;
a4 = -(c4 - b4);
System.out.println(a4); // result 7

Операция прибавления единицы к числу

int a5, b5 = 5, c5 = -2;
a5 = -c5;
a5++; // a5 = a5 + 1;
System.out.println(a5); // result 3

Вместо длинной конструкции можно применять короткую:
длинная: a5 = a5 + 1;
короткая: a5++;

Операция a5++ это унарная операция, и плюсы можно ставить как слева так и справа от имени переменной:

a5++

++a5

Приоритет префиксной и постфиксной операции ++

Постфиксная операция ++

int a6, b6 = 5, c6 = -2;
a6 = -c6;
b6 = a6++; // a6 = a6 + 1;
System.out.println(b6); // result 2

Приоритет операций:
Сначала выполняется b6 = a6, затем a6++
Присвоение переменной b6 значения переменной a6 происходит до того, как a6 будет увеличена на 1

Префиксная операция ++

int a7, b7 = 5, c7 = -2;
a7 = -c7;
b7 = ++a7; // a7 = a7 + 1 | b7 = a7
System.out.println(b7); // result 3

Приоритет операций:

Сначала выполняется увеличение на 1 значения переменной a, затем b7 = a7

Такой же алгоритм выполнения приоритетов операций с оператором --
